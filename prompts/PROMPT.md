- 适配手机和电脑

### 2. 核心游戏 (game.html)
基于现有代码重构, 在 game.html 中实现:

#### 地图系统
- 30x20 网格, 草地(绿)/山路(灰褐)/池塘(蓝)
- 池塘不能通行, 山路减速
- 洞口明显: 深色椭圆, 有立体感
- 生成5~8个洞, 随机分布在草地

#### 洞穴系统
- 两种洞: 连通洞(多个出口, 用绿色光晕标记) 和 死路洞(单出口, 无光晕)
- 不同虫子的洞外观不同 (土洞/草洞/树洞/花洞/网洞/粪洞)
- 虫子进洞后计时 (5~15秒), 超时自动出洞
- 虫子出洞后需要冷却才能再次进洞
- 连通洞的虫子可以从另一个出口出来

#### 铲子系统
- 站在洞口按 Q 挖洞
- 死路洞+有虫: 额外+30分, 体力-10
- 死路洞+无虫: 体力-25
- 连通洞: 破坏洞穴, 体力-15

#### 虫子行为 AI
- 蚂蚱: 会跳跃移动 (跳2~3格距离), 落地停一下再跳
- 螳螂: 静止时难发现(半透明), 玩家靠近才动
- 独角仙: 移动慢但耐力好, 飞得久
- 蝴蝶: 飘忽飞行路线, 正弦波轨迹
- 知了: 停在树上(地图边缘), 发出"知了"文字提示
- 蜘蛛: 织网区域, 在网内移动快, 出网慢, 有毒
- 屎壳郎: 推粪球移动, 接触中毒, 扣分陷阱

#### 图片生成 (通过API)
使用 gpt-image-2 或其他可用模型生成:
- 小朋友角色: 戴草帽的男孩/女孩, 可爱卡通风格
- 7种虫子: 写实但卡通风, 非像素风格
- 洞的贴图: 有立体感的洞穴
- 尺寸: 64x64 PNG 带透明背景

生成方式: 
- 在 game.js 中写一个 generateAssets() 函数
- 调用 API: POST https://www.uocode.com/v1/images/generations
  - Header: Authorization: Bearer sk-DWgu0g50RFo2yv2wvNDFATrDApBfFAJNOww5BtvhWrKy2yrF
  - Body: {"model": "dall-e-3", "prompt": "...", "n": 1, "size": "1024x1024"}
- 把返回的图片URL下载到 assets/ 目录
- 如果API不可用, 用 Canvas 绘制替代方案 (但必须好看)

#### 音效
- 使用 Web Audio API 生成简单音效 (不需要外部文件)
- 抓虫成功: 清脆"叮"声
- 挖洞: 低沉"咚"声
- 中毒: 警报音
- 背景: 夏日虫鸣环境音 (低频循环)
- 知了叫声: 周期性"知了知了"音

#### 昆虫图鉴 (collection.html)
- 显示所有7种昆虫的卡片
- 每个卡片: 图片/名字/稀有度/分数/是否已捕获
- 已捕获显示捕获数量
- 毛玻璃风格, 同首页一致

#### 排行榜 (leaderboard)
- 用 localStorage 存储最高分
- 显示前5名
- 每次游戏结束更新

#### 视觉风格
- 适合6-12岁, 色彩鲜艳活泼
- 不要像素风
- 毛玻璃UI元素
- 圆角, 柔和阴影
- 字体大而清晰
