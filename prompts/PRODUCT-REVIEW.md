## 二、角色自定义系统 (方案)

这是最核心的需求。我设计一个**Canvas实时绘制+自由搭配**的系统:

### 组件面板

```
┌──────────────────────────────────┐
│       👤 创建你的角色            │
│                                  │
│     ╔══════════════════╗         │
│     ║  实时角色预览     ║         │ ← Canvas绘制
│     ║  (搭配即时显示)   ║         │
│     ╚══════════════════╝         │
│                                  │
│  ┌───── 自定义选项 ─────────┐   │
│  │ 🧢 帽子: [棒球帽][草帽]  │   │
│  │        [探险帽][空]      │   │
│  │ 👧 发型: [短发][辫子]   │   │
│  │        [卷发][寸头]     │   │
│  │ 👕 上衣: ■ ■ ■ ■ ■ ■   │   │ ← 颜色色块
│  │ 👖 裤子: ■ ■ ■ ■ ■ ■   │   │
│  │ 👟 鞋子: ■ ■ ■ ■ ■ ■   │   │
│  │ 📛 名字: [________]     │   │
│  └────────────────────────┘   │
│                                  │
│     [ ✅ 命名, 出发! ]          │
└──────────────────────────────────┘
```

### 技术方案

用 Canvas 在选人界面实时绘制:
1. 画身体轮廓(衣服颜色)
2. 画头+发型
3. 画帽子
4. 画裤子+鞋子颜色
5. 生成 dataURL → 作为游戏中的角色精灵

不需要 gpt-image-2 每次生成，不花 token，实时响应。

### 色彩选择

提供 12 种预设色块:
```
红 🔴 橙 🟠 黄 🟡 绿 🟢 蓝 🔵 紫 🟣
粉 🩷 棕 🟤 灰 ⚪ 黑 ⚫ 青 🩵
```

## 五、我的建议

嘉哥, 我觉得现在游戏的核心玩法已经成型了: 大世界+7种虫AI+洞穴+铲子+音效+BGM。当前最大的短板是**角色创建**和**视觉品质**。

我建议:

1. **先做角色自定义** — 这是玩家进游戏的第一印象, 也是最有成就感的部分
2. **再做首页美化** — 第二印象
3. **最后打磨游戏体验** — 连击/特效/引导

每个步骤我出完方案你审, 你点头我再叫 Codex 干活。不再闷头码了。
