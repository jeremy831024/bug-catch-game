# 抓虫大冒险 Unity 沙盒生存玩法规划

## 1. 设计目标

游戏要从“抓更多虫”升级为“在有限资源和动态环境里，做出最值的一次野外探索”。

核心体验：

- 玩家有明确目标
- 玩家持续面临取舍
- 玩家能发现新东西
- 玩家能通过成长降低风险

## 2. 四个乐趣支柱

### 2.1 探索

每次进入地图都要有未知感。

可实现内容：

- 随机地图区块
- 区域差异：草地、沼泽、洞穴、林地、湖边
- 天气和昼夜改变可见性
- 稀有虫和补给点随机出现

### 2.2 取舍

玩家不应该什么都能带、什么都能做。

可实现内容：

- 背包容量有限
- 食物、解毒药、手电、陷阱工具互相抢占格子
- 高价值虫通常在更危险区域
- 继续深入或返回安全区之间形成选择

### 2.3 发现

收集本身要有反馈。

可实现内容：

- 图鉴首次点亮
- 稀有虫特殊捕获提示
- 区域发现记录
- 夜间限定、雨后限定、雾天限定虫子

### 2.4 压力

压力要温和但持续。

可实现内容：

- 体力持续消耗
- 中毒持续扣体力并减速
- 夜晚视野缩小
- 捕食者巡逻
- 蜘蛛网和泥坑限制路线

## 3. 最推荐优先做的系统

### 3.1 安全营地

营地是玩家每局的心理锚点。

功能：

- 休息恢复体力
- 整理背包
- 查看图鉴
- 接每日任务
- 出发前选择道具

### 3.2 任务板

任务给玩家明确方向。

任务类型：

- 抓指定虫
- 探索指定区域
- 带回指定数量样本
- 夜晚限定任务
- 天气限定任务

### 3.3 地图分区

不同区域提供不同风险收益。

区域建议：

- 草地：安全，普通虫多
- 林地：知了、独角仙、树栖虫
- 沼泽：滑、慢、补给少，稀有虫多
- 洞穴：视野差，夜行虫和危险更高
- 湖边：补给和特殊虫，但移动路线受限

### 3.4 随机生态事件

事件让每局不一样。

事件示例：

- 雨后独角仙出现
- 夜晚知了群鸣
- 雾天稀有虫靠近湖边
- 青蛙巡逻路线变化
- 蜘蛛网刷新在捷径上

### 3.5 抓捕反馈

抓虫必须有明确成就感。

反馈组合：

- 捕虫网挥动动画
- 粒子闪光
- 浮动分数
- 音效
- 图鉴点亮
- 稀有虫慢动作或镜头轻震

## 4. 道具链设计

### 4.1 食品

作用：

- 恢复体力
- 随机生成
- 是探索续航核心

设计重点：

- 食物不能太多，否则没有压力
- 食物位置要引导玩家探索

### 4.2 解毒药

作用：

- 随身携带后降低中毒伤害
- 不是完全免疫

设计重点：

- 让玩家在“带食物还是带解毒药”之间取舍

### 4.3 加速靴

作用：

- 临时提速
- 适合逃离捕食者或追稀有虫

### 4.4 隐身叶

作用：

- 短暂躲避捕食者
- 对虫子可见性可做轻微影响

### 4.5 陷阱工具

作用：

- 放置后自动捕虫
- 适合守稀有虫路线

设计重点：

- 需要消耗背包空间
- 放错位置会浪费

### 4.6 手电筒

作用：

- 夜晚提升可视范围
- 洞穴探索必备

设计重点：

- 可设置电量或持续时间

## 5. 虫子体验规划

更完整的虫子生态、声音提示和近距离感知规则见 [unity-insect-ecology-and-sensing.md](./unity-insect-ecology-and-sensing.md)。

### 5.1 普通虫

目标：

- 教会玩家基础抓捕
- 提供稳定收益

例子：

- 蚂蚱
- 普通蝴蝶
- 小甲虫

### 5.2 行为虫

目标：

- 让抓捕方式产生变化

例子：

- 逃跑型：靠近就跑
- 伪装型：不移动时难发现
- 飞行型：需要判断高度和方向
- 树栖型：只能停在树上

### 5.3 危险虫

目标：

- 给玩家风险判断

例子：

- 蜘蛛：织网减速
- 毒虫：触碰中毒
- 攻击型昆虫：短距离冲刺

### 5.4 稀有虫

目标：

- 形成探索和冒险动机

出现条件：

- 夜晚
- 雨后
- 危险区域
- 特定地形附近

## 6. 分期路线

### Phase 1：可玩核心

目标：证明第三人称 3D 抓虫 + 体力生存成立。

范围：

- 角色移动
- 基础地图
- 体力消耗
- 食物补给
- 3 种虫
- 抓捕反馈
- 简单图鉴

验收：

- 玩家能玩完一局
- 体力归零会结束
- 抓虫能获得明确反馈

### Phase 2：生存压力

目标：让玩家开始做资源取舍。

范围：

- 背包容量
- 解毒药
- 蜘蛛网
- 毒蘑菇
- 安全营地
- 每日任务

验收：

- 玩家会主动选择携带物
- 玩家会考虑什么时候返回营地

### Phase 3：动态环境

目标：让每局体验不同。

范围：

- 昼夜循环
- 雨天
- 雾天
- 夜行虫
- 天气限定事件

验收：

- 玩家能感受到白天、夜晚、雨天、雾天玩法差异

### Phase 4：生态与长期目标

目标：提升耐玩度。

范围：

- 捕食者
- 稀有虫
- 区域解锁
- 角色成长
- 排行榜
- 成就

验收：

- 玩家有继续收集和反复游玩的理由

## 7. 风险控制

### 7.1 不要过早扩大地图

先做小地图密度，确认每分钟都有选择和事件。

### 7.2 不要过早堆美术

先用占位符验证玩法，再替换模型、贴图、动画。

### 7.3 不要把生存压力做太硬

目标是轻度沙盒生存，不是高惩罚硬核生存。

### 7.4 每个系统都要服务取舍

如果一个系统没有带来探索、取舍、发现或压力，就应该推迟。
